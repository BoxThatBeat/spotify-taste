## Critical Security Requirements
- OAuth state parameter MUST be generated (crypto.randomBytes), stored in session, and validated on callback
- Client secret NEVER exposed to frontend
- Tokens stored server-side only (req.session.tokens.userA, req.session.tokens.userB)

## Edge Cases (from CONTEXT.md)
- Duplicate account detection: Block if User B authorizes same Spotify ID as User A
- Session timeout: Clear session if too much time between authorizations (suggest 15 minutes)
</context>

  <action>
Create routes/auth.js with SpotifyWebApi configuration and three route handlers:

**Setup:**
```javascript
const SpotifyWebApi = require('spotify-web-api-node');
const crypto = require('crypto');

const spotifyApi = new SpotifyWebApi({
  clientId: process.env.SPOTIFY_CLIENT_ID,
  clientSecret: process.env.SPOTIFY_CLIENT_SECRET,
  redirectUri: process.env.SPOTIFY_REDIRECT_URI
});

const SCOPES = ['user-top-read']; // AUTH-04 requirement
```

**GET /login/userA:**
- Generate random state: `crypto.randomBytes(16).toString('hex')`
- Store in session: `req.session.oauthState = state; req.session.currentUser = 'userA'`
- Create authorization URL: `spotifyApi.createAuthorizeURL(SCOPES, state)`
- Redirect user to Spotify authorization URL

**GET /login/userB:**
- Check if userA tokens exist: `if (!req.session.tokens.userA)` → return 400 error "User A must authorize first"
- Generate random state: `crypto.randomBytes(16).toString('hex')`
- Store in session: `req.session.oauthState = state; req.session.currentUser = 'userB'`
- Create authorization URL: `spotifyApi.createAuthorizeURL(SCOPES, state)`
- Redirect user to Spotify authorization URL

**GET /callback:**
- Validate state parameter: `if (req.query.state !== req.session.oauthState)` → return 400 "Invalid state parameter"
- Handle authorization errors: `if (req.query.error)` → return 400 with friendly message
- Exchange code for tokens:
  ```javascript
  const data = await spotifyApi.authorizationCodeGrant(req.query.code);
  const { access_token, refresh_token, expires_in } = data.body;
  ```
- Get user profile to check for duplicates:
  ```javascript
  spotifyApi.setAccessToken(access_token);
  const profile = await spotifyApi.getMe();
  const spotifyId = profile.body.id;
  ```
- Duplicate account check:
  ```javascript
  const currentUser = req.session.currentUser;
  const otherUser = currentUser === 'userA' ? 'userB' : 'userA';
  const otherTokens = req.session.tokens[otherUser];
  
  if (otherTokens && otherTokens.spotifyId === spotifyId) {
    return res.status(400).send('This account already authorized. User B needs a different account.');
  }
  ```
- Store tokens in session:
  ```javascript
  req.session.tokens[currentUser] = {
    accessToken: access_token,
    refreshToken: refresh_token,
    expiresAt: Date.now() + (expires_in * 1000),
    spotifyId: spotifyId,
    displayName: profile.body.display_name
  };
  ```
- Clear OAuth state: `delete req.session.oauthState; delete req.session.currentUser`
- Redirect to success page: `res.redirect('/?status=success')`

**Error handling:** Wrap all async handlers in try/catch, return friendly error messages (not raw Spotify API errors).

**Mount routes in server.js:**
```javascript
const authRoutes = require('./routes/auth');
app.use('/', authRoutes);
```

<task type="auto">
  <name>Task 1: Create Spotify data fetching route with token refresh</name>
  <files>routes/spotify-data.js</files>
  <action>
Create `routes/spotify-data.js` with GET `/api/fetch-data` endpoint that:

**Input validation:**
- Accepts `timeRange` query parameter (short_term, medium_term, long_term)
- Defaults to medium_term (6 months) if not provided
- Returns 400 if both users not authorized (check req.session.tokens.userA and userB)

**Token refresh logic:**
- Before each Spotify API call, check if token expired (compare Date.now() vs expiresAt)
- If expired: call spotifyApi.refreshAccessToken() with refresh token, update session with new access token and expiry
- Use refreshed token for subsequent API calls
- If refresh fails: return 401 error with message "Session expired, please restart authorization"

**Parallel data fetching:**
Use Promise.all() to fetch 4 data types concurrently:
1. User A top artists: spotifyApi.setAccessToken(userA token), spotifyApi.getMyTopArtists({time_range, limit: 50})
2. User A top tracks: spotifyApi.getMyTopTracks({time_range, limit: 50})
3. User B top artists: spotifyApi.setAccessToken(userB token), spotifyApi.getMyTopArtists({time_range, limit: 50})
4. User B top tracks: spotifyApi.getMyTopTracks({time_range, limit: 50})

**Rate limit handling:**
- Wrap all Spotify API calls in try/catch
- If error.statusCode === 429 (rate limit): Extract Retry-After header, wait that many seconds, retry same request
- Use exponential backoff: 1st retry = Retry-After, 2nd retry = 2x, 3rd retry = 4x, max 3 retries
- If max retries exceeded: return 503 error with message "Spotify API rate limited, try again in a few minutes"

**Response format:**
Return JSON with structure:
```json
{
  "userA": {
    "displayName": "...",
    "topArtists": [{id, name, images, genres}, ...],
    "topTracks": [{id, name, album: {images}, artists: [{name}]}, ...]
  },
  "userB": { ... same structure ... },
  "timeRange": "medium_term"
}
```

**Error handling:**
- Catch all errors, log full technical details to console.error()
- Return appropriate HTTP status (400/401/429/500) with error message in response body
- Include stack trace in development mode (check NODE_ENV)

**Implementation notes:**
- Reuse SpotifyWebApi client from routes/auth.js pattern (import spotify-web-api-node)
- Configure client with same SPOTIFY_CLIENT_ID, SPOTIFY_CLIENT_SECRET, SPOTIFY_REDIRECT_URI from env
- Token refresh updates both access_token and expires_in, store updated expiresAt = Date.now() + (expires_in * 1000)
